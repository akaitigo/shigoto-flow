# PRD: Shigoto-Flow

## 概要
日報・週報・月報の作成を自動化する中小企業向けレポート自動生成ツール。
Googleカレンダー・Slack・GitHub・Gmailから活動データを自動集約し、日報テンプレートに流し込む。
ユーザーは自動生成された内容を確認・修正して送信するだけで日報が完成する。

## 背景・課題
- 中小企業の従業員は毎日15-30分を日報作成に費やしている
- 活動情報は各ツールに散在しており、手動で集約する手間がある
- 週報・月報は日報の再集約で、さらに時間を浪費する
- 日報の質はモチベーションに依存し、記載漏れが頻発する

## ユーザーペルソナ
- **メインユーザー**: 中小企業の一般従業員（営業/エンジニア/管理部門）
- **管理者**: チームリーダー/上長（レポート受信・確認者）

## MVP機能

### F1: 外部データソース自動集約
- Googleカレンダーから当日の予定を取得
- Slackから当日の自分の投稿を取得
- GitHubから当日のコミット・PR・Issueを取得
- Gmailから当日の送受信サマリーを取得
- OAuth2による安全な認証フロー

### F2: 日報自動生成・編集・送信
- 集約データを日報テンプレートに自動流し込み
- リッチテキストエディタでの確認・修正
- テンプレートのカスタマイズ（セクション追加/削除/並べ替え）
- 日報の送信（メール/Slack通知）
- 日報履歴の閲覧・検索

### F3: 週報・月報自動要約
- 日報データからClaude APIを使って週報を自動生成
- 週報データから月報を自動生成
- 要約のカスタマイズ（重点項目/除外項目の設定）
- 上長への自動送信設定

## 技術スタック
| レイヤー | 技術 | 用途 |
|---------|------|------|
| フロントエンド | Next.js (TypeScript) | レポート編集UI/設定ダッシュボード |
| バックエンド | Go | API/データ集約/レポート生成 |
| データベース | PostgreSQL | レポート・データソース・テンプレート |
| 認証 | OAuth2 | Google/Slack/GitHub連携 |
| AI | Claude API | 日報→週報→月報の要約 |
| インフラ | GCP Cloud Run | コンテナデプロイ |

## 非機能要件
- レスポンスタイム: API 500ms以内（要約生成除く）
- データ保持: レポートは無期限保持
- セキュリティ: OAuth2トークンは暗号化して保存
- 可用性: Cloud Runのオートスケーリング
